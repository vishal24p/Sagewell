## Rules

- Record decisions that affect architecture, security, data model,
  deployment, or workflow behavior.
- Keep each ADR short enough to read in a few minutes.
- Include alternatives and consequences.
- Mark superseded decisions instead of rewriting history.
- Out-of-V1 concepts require their own ADR if reintroduced.
