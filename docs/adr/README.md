# Architecture Decision Records

Architecture Decision Records capture durable decisions, rejected alternatives, and consequences.

## Rules

- Record decisions that affect architecture, security, data model, deployment, or workflow behavior.
- Keep each ADR short enough to read in a few minutes.
- Include alternatives and consequences.
- Mark superseded decisions instead of rewriting history.
